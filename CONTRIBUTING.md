## Code of Conduct

This project and everyone participating in it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [project_email@example.com].

## Getting Started

Contributions to this project are made through GitHub Issues and Pull Requests.

### Issues

- Feel free to open an issue for any bugs you've found or features you'd like to see implemented.
- Before opening a new issue, please check if a similar issue already exists.
- When opening an issue, please provide as much detail as possible, including steps to reproduce for bugs.

### Pull Requests

1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Coding Guidelines

- Follow PEP 8 style guide for Python code.
- Use meaningful variable and function names.
- Write comments for complex logic.
- Keep functions small and focused on a single task.
- Use type hints where applicable.

## Commit Message Guidelines

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification. This leads to more readable messages that are easy to follow when looking through the project history. 

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line

## Testing

- Write unit tests for new code you create.
- Ensure all tests pass before submitting a pull request.
- We use pytest for our testing framework.

## Documentation

- Keep README.md and other documentation up to date as you make changes.
- Comment your code where necessary.
- If you're adding a new feature, please add documentation explaining how to use it.

## Community

- Feel free to join our [community chat](https://example.com/chat) to discuss the project with other contributors.
- We also have regular video calls every other Thursday. Check the [community calendar](https://example.com/calendar) for details.

Remember, contributions to this project should be fun and educational. Don't hesitate to ask questions if something's not clear. We're all here to learn and create something awesome together!

Thank you for your interest in contributing to the Kubernetes-Driven Digital Aquarium project. We look forward to your contributions!
